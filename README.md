# Frontend_pictures: Інструкція з розгортання

## 1. Вимоги до оточення
Для стабільної роботи проекту необхідно мати:

+ Docker (версія 20.10+)
+ Docker Compose (версія 2.0+)
+ ОС: Linux, macOS або Windows (з встановленим Docker Desktop)
+ Вільні порти: 8080 (для веб-інтерфейсу) та 5432 (якщо плануєш прямий доступ до БД).

## 2. Структура проекту

```
image_server/
├── app.py                # Python HTTP бекенд (основна логіка)
├── requirements.txt      # Залежності Python (Pillow)
├── Dockerfile            # Інструкція для створення Docker-образу
├── compose.yaml          # Конфігурація Docker Compose (2 сервіси)
├── nginx.conf            # Конфігурація веб-сервера Nginx
├── static/               # Фронтенд (HTML, CSS, JS)
│   ├── index.html        # Головна сторінка
│   ├──upload.html        # Сторінка завантаження
│   ├──images.html        # Сторінка списку зображень
│   └── file_style_css/
│   │    ├── css/         # Стилі (index.css, reset.css, imager.css, upload.css)│   │
│   ├── js/               # Скрипти (index.js, upload.js, images.js, common.js)
│   ├── img/              # Зображення для дизайну
├── images/               # Папка для завантажених зображень (Docker volume)
├── logs/                 # Папка для логів (Docker volume)
├── backup.sh             # Збереження зображень до бази даних
├── restore.sh            # Повернення збереженої бази даних (на випадок збійної ситуації)
└── services.py           # Перевірка файлів ввід/вивід та базу даних
```

## 3. Команди для керування
### Запуск проекту:
Ця команда збирає образи та запускає всі контейнери у фоновому режимі:

```bash
docker-compose up -d
```

### Зупинка проекту:
Зупиняє контейнери, але зберігає дані в базі:

```bash
docker-compose stop
```

### Повне видалення:
Зупиняє та видаляє контейнери (дані в Volume залишаться):

```bash
docker-compose down
```

### Перегляд логів (якщо щось не працює):

```bash
docker-compose logs -f app
```
## 4. Опис маршрутів 
(Routes)Веб-інтерфейс (Frontend):

### Веб-інтерфейс (Frontend)

| Маршрут | Файл | Функціональність |
| :--- | :--- | :--- |
| `/` | `index.html` | Головна сторінка з вітальним екраном. |
| `/upload` | `upload.html` | Форма для Drag-and-Drop завантаження файлів. |
| `/images-list` | `images.html` | Галерея з таблицею, прев'ю та пагінацією. |

### API (Backend)

| Ендпоінт | Метод | Опис |
| :--- | :--- | :--- |
| `/api/upload` | `POST` | Приймає файл, зберігає в `uploads/` та робить запис у БД. |
| `/api/images` | `GET` | Повертає JSON-список зображень з підтримкою сторінок (`?page=1`). |
| `/delete/<id>` | `POST` | Видаляє запис про зображення з бази та сам файл із диска. |
| `/images/<name>` | `GET` | Прямий доступ до завантаженого файлу (через Nginx). |
